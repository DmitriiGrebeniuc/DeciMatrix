Общие правила проекта:

Мы делаем мобильное приложение на Expo + React Native + TypeScript.

Продукт: мобильное приложение для принятия решений через взвешенную матрицу.

Главный UX-принцип:
- один экран - одно действие;
- минимум ручного ввода;
- слайдеры для важности критериев;
- цветная шкала оценки вместо ввода цифр;
- результат должен быть понятным и визуальным.

Стек:
- Expo
- React Native
- TypeScript
- Expo Router
- Zustand
- AsyncStorage
- @react-native-community/slider

Не добавлять:
- backend;
- авторизацию;
- Supabase;
- push notifications;
- dark mode;
- платные функции;
- экспорт PDF/Excel;
- AI-функции;
- лишние библиотеки без необходимости.

После каждой задачи:
- проверь TypeScript;
- проверь импорты;
- проверь, что приложение запускается;
- кратко напиши, какие файлы изменены;
- кратко напиши, что проверено.
